Functional Testing

Assures that the developed software meets the requirements.

They should not be our unique tests.

Ideal testing pyramid

The base of the pyramid are the unit tests

  • Simple
  • Fast
  • Robust

Another vision

The inverted pyramid

The base of the pyramid is unstable.

Most of the tests are manual tests

  • Complex
  • Slow
  • Fragile

Maintain a lot of end to end tests is hard

  • Long-running
  • Many resources
  • Paths hard to test
  • Uncertainity
  • Coupled
  • Not suitable for refactoring
They may take a lot of time to execute They may need a lot of resources There are patha hard to test When they fail, we don't know exactly where o why they fail. Or not inmediatly. They are so coupled with the environment They don't give the same sense of security when refactoring.

CasperJS

CasperJS is an open source navigation scripting & testing utility written in Javascript and based on PhantomJS.

It eases the process of defining a full navigation scenario and provides useful high-level functions, methods & syntactic sugar for doing common tasks.

PhantomJS

PhantomJS is a headless WebKit scriptable with a JavaScript API. It has fast and native support for various web standards: DOM handling, CSS selector, JSON, Canvas, and SVG.

Evaluate

Evaluates an expression in the current page DOM context.

Think of the evaluate() method as a gate between the CasperJS environment and the one of the page you have opened; everytime you pass a closure to evaluate(), you're entering the page and execute code as if you were using the browser console.

example

casper.start('http://casperjs.azurewebsites.net/');

casper.then(function()
{
	this.echo(this.evaluate(function()
	{
		return document.title;
	}));
});

exercise

Change the page body background color and then, make a capture.

Solution

casper.start('http://casperjs.azurewebsites.net/');

casper.then(function()
{
	this.evaluate(function()
	{
		$("body").css("background-color", "red");
		return;
	});
});

casper.then(function(){
	this.capture('redBody.png');
});

injecting scripts

We can inject scripts into the remote DOM in every response by setting the clientScripts option of CasperJS

var casper = require('casper').create({
    clientScripts:  [
        'includes/jquery.js',      // These two scripts will be injected in remote
        'includes/underscore.js'   // DOM on every request
    ]
});

Testing

The basic structure

var casper = require('casper').create();

casper.start('http://casperjs.azurewebsites.com');

casper.then(function(){
	this.test.pass('SpainJS is a great conference');
	this.test.fail('I am not going to talk with anybody these three days');
});

casper.run(function(){
	this.test.done(2);
	this.test.renderResults(true);
});

Basic asserts

Assert

Asserts that the provided condition strictly resolves to a boolean true.

this.test.assert(this.getCurrentUrl() === url, 'url is the one expected');

AssertNot

Asserts that the passed subject resolves to some falsy value.

casper.test.assertNot(false, "Universe is still operational");

AssertTruthy

Asserts that a given subject is truthy.

casper.test.assertTruthy(true, "The workshop is going well");

AssertFalsy

Asserts that a given subject is falsy.

casper.test.assertFalsy(true, "I'm bored");

AssertEquals

Asserts that two values are strictly equals.

this.test.assertEquals(this.getCurrentUrl(), url, 'url is the one expected');

AssertNotEquals

Asserts that two values are not strictly equals.

this.test.assertNotEquals(Barça == Madrid, 'Depends who you ask');

more advanced asserts

assertExists

Asserts that an element matching the provided selector expression exists in remote DOM environment.

this.test.assertExists('form[name="gs"]', 'google.fr has a form with name "gs"');

assertDoesntExist

Asserts that an element matching the provided selector expression doesn't exists within the remote DOM environment.

this.test.assertDoesntExist('form[name="gs"]', 'google.fr has a form with name "gs"');

assertEval

Asserts that a code evaluation in remote DOM strictly resolves to a boolean true.

casper.start('http://www.google.fr/', function() {
    this.test.assertEval(function() {
        return document.querySelectorAll('form').length > 0;
    }, 'google.fr has at least one form');
    this.test.assertEval(function(title) {
        return document.title === title;
    }, 'google.fr title is "Google"', 'Google');
});

assertField

Asserts that a given form field has the provided value:

casper.start('http://www.google.fr/', function() {
    this.fill('form[name="gs"]', { q: 'plop' }, false);
    this.test.assertField('q', 'plop');
});

assertHttpStatus

Asserts that current HTTP status code is the same as the one passed as argument.

this.test.assertHttpStatus(200, 'google.fr is up');

assertVisible

Asserts that the element matching the provided selector expression is visible.

this.test.assertVisible('h1');

assertNotVisible

Asserts that the element matching the provided selector expression is not visible.

this.test.assertNotVisible('h6');

assertSelectorExists

Asserts that at least an element matching the provided selector expression exists in remote DOM.

this.test.assertSelectorExists('form[name="gs"]', 'google.fr provides a form');

assertSelectorHasText

Asserts that given text exists in the provided selector.

this.test.assertSelectorHasText('title', 'Google');

assertSelectorDoesntHaveText

Asserts that given text does not exist in the provided selector.

this.test.assertSelectorDoesntHaveText('title', 'Yahoo!');

assertTextExists

Asserts that body plain text content contains the given string.

this.test.assertTextExists('google', 'page body contains "google"');

assertTextDoesntExist

Asserts that body plain text content doesn't contain the given string.

this.test.assertTextDoesntExist('bing', 'page body does not contain "bing"');

assertTitle

Asserts that title of the remote page equals to the expected one.

this.test.assertTitle('Google', 'google.fr has the correct title');

Using different files for testing

We could split our tests in serveral files. With two conditions:

Not to create a new Casper instance in a split test file.

To call the Tester.done() method when all the tests contained in a single file have been executed.

Sample

File 1

casper.test.pass('Suite 1 passing test');
casper.test.fail('Suite 1 failing test');
casper.test.done(2);

File 2

casper.test.pass('Suite 2 passing test');
casper.test.fail('Suite 2 failing test');
casper.test.fail('Suite 2 another failing test');
casper.test.done(3);

And now call:

casperjs test tests --xunit=testsResults.xml
